Travel Link Project

Stack: React, Python FastAPI, MySQL 8.0, AWS RDS  |  Role: Backend & DB Engineer
1. Project Overview

Google Gemini API를 활용하여 개인화된 여행 경로와 맛집을 추천하고, 동행자를 모집하는 플랫폼입니다.
React로 직관적인 UI를 구성하고, Python FastAPI로 고성능 비동기 서버를 구축했습니다.

2. [The Architect] Hybrid DB Modeling

비정형 데이터(AI 응답)와 정형 데이터(회원/예약)를 효율적으로 처리하기 위해 Hybrid 설계를 도입했습니다.

  • Problem: 여행 일정은 구조가 가변적이라 정규화 시 수백 번의 JOIN 발생 및 스키마 경직성 확인.
  • Solution: MySQL 8.0 JSON Type을 도입하여 가변 데이터를 반정규화하여 저장.
  • Impact: 스키마 변경 없이 AI 모델 업데이트에 유연하게 대처, 조회 쿼리 복잡도 70% 감소.
3. Entity Relationship Diagram (ERD)

* Plans(여행계획)와 Details(상세일정)의 관계 및 JSON 컬럼 활용

4. [The Problem Solver] Query Tuning

여행지 리스트 조회 시 발생하던 Slow Query (3.8s → 0.03s)를 개선했습니다.

  • Analysis: EXPLAIN 분석 결과 type: ALL(Full Table Scan) 및 Using filesort 발생 확인.
  • Optimization: 카디널리티를 고려한 (region, status, created_at) 복합 인덱스 설계.
  • Metrics: 쿼리 속도 약 120배 단축 달성.
5. [The Engineer] Concurrency Control

선착순 참여 기능에서 발생한 초과 예약(Race Condition) 문제를 해결했습니다.

  • Situation: 마지막 1자리 남았을 때 동시 요청 시 정원 초과 저장 문제.
  • Solution: 트랜잭션 격리 수준을 고려하여 비관적 락(Pessimistic Lock, FOR UPDATE) 적용.
  • Verification: JMeter 부하 테스트 결과 정합성 100% 유지.
