About Me

"데이터가 흐르는 가장 효율적인 길을 찾습니다."

애플리케이션의 성능은 결국 데이터베이스에서 결정된다고 믿습니다. 백엔드 개발 과정을 통해 데이터 모델링과 쿼리 최적화의 중요성을 깊이 체감했습니다.

SQLD 자격증을 취득하며 관계형 데이터베이스의 이론적 토대를 다졌으며, 실무 프로젝트를 통해 동시성 제어(Concurrency Control)와 인덱스 튜닝을 통해 성능을 개선하는 과정에 큰 흥미를 느꼈습니다.

비즈니스의 요구사항을 정확한 데이터 모델로 변환하고, 대용량 트래픽 상황에서도 데이터 무결성을 지키는 믿음직한 DB 엔지니어로 성장하고 싶습니다.

Main Project

✈️ Travel Link (AI 기반 여행 계획 공유 플랫폼)

Period: 2025.03 ~ 2025.12  |  Role: DB Architect & Backend Developer  |  Category: DB Engineering
1. 프로젝트 개요
  • 서비스명: Travel Link
  • 기술 스택: Python FastAPI, SQLAlchemy, MySQL 8.0, AWS RDS
  • 나의 역할: 데이터베이스 설계 및 백엔드 API 개발 총괄
2. 핵심 DB 설계 (JSON Modeling)

AI가 생성하는 가변적인 여행 일정 데이터를 처리하기 위해 RDBMS와 NoSQL(JSON)의 장점을 결합한 Hybrid 설계를 도입했습니다.

  • 문제: 일정을 정규화할 경우 조회 시 수백 개의 Row를 JOIN해야 하는 오버헤드 발생.
  • 해결: MySQL 8.0 JSON Type 도입 및 Check Constraint로 무결성 확보.
  • 성과: 상세 조회 쿼리 단순화 및 응답 속도 최적화.
3. 동시성 제어 (Concurrency Control)

선착순 모집 시 발생하는 초과 예약(Over-booking) 문제를 해결했습니다.

  • 상황: 정원 1명 남았을 때 동시 요청 발생.
  • 해결: 비관적 락(Pessimistic Lock, FOR UPDATE)을 적용하여 트랜잭션 격리 수준을 엄격하게 관리.
  • 결과: JMeter 부하 테스트 결과, 동시성 이슈 0건 방어 성공.
4. 성능 최적화

메인 리스트 조회 시 발생하는 FileSort를 제거하여 성능을 개선했습니다.

  • 해결: (destination, created_at) 복합 인덱스(Composite Index) 적용.
  • 성과: 조회 속도 0.45s → 0.03s로 약 15배 개선.
