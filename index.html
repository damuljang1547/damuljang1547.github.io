Travel Link Project

Period: 2025.03 ~ 2025.12  |  Role: DB Architect & Backend Developer
1. 프로젝트 개요

사용자의 취향을 분석하여 여행 경로를 추천하고 공유하는 플랫폼입니다.

  • 기술 스택: Python FastAPI, SQLAlchemy, MySQL 8.0, AWS RDS
  • 핵심 기여: DB 스키마 설계, 트랜잭션 관리, 성능 최적화
2. 핵심 DB 설계 (JSON Modeling)

AI가 생성하는 비정형 여행 데이터 처리를 위해 Hybrid 설계를 도입했습니다.

  • 문제: 정규화 시 수백 개의 Row를 JOIN해야 하는 조회 성능 저하 발생.
  • 해결: MySQL 8.0 JSON Type 도입 및 Check Constraint로 무결성 확보.
  • 성과: 복잡한 조회를 단일 쿼리로 처리하여 응답 속도 최적화.
3. 동시성 제어 (Concurrency Control)

선착순 모집 기능에서 발생하는 초과 예약(Over-booking) 문제를 해결했습니다.

  • 상황: 마지막 1자리 예약 시 다수의 요청이 동시에 들어오는 Race Condition 발생.
  • 해결: 비관적 락(Pessimistic Lock, FOR UPDATE)을 적용하여 데이터 정합성 보장.
  • 결과: JMeter 부하 테스트 결과, 동시성 오류 0% 달성.
4. 성능 최적화 (Query Tuning)

메인 리스트 조회 시 발생하는 FileSort를 제거했습니다.

  • 해결: (destination, created_at) 복합 인덱스(Composite Index) 설계 및 적용.
  • 성과: 조회 속도 0.45s → 0.03s로 약 15배 개선.
